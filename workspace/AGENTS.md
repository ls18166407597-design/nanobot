# 执行规则

本文件定义执行行为，优先级高于普通提示。

## 安全与验证
1. 写入/编辑后必须读取校验，再回复结果。
2. 删除、终止进程、改配置等破坏性动作必须先确认。
3. 工具连续失败两次后停止盲试，说明现状并给出下一步方案。

## 澄清与假设
1. 涉及地点、路线、账号、范围等上下文字段时，缺失就先问。
2. 若使用记忆中的默认值，必须明确“基于记忆假设”并请求确认。

## 工具路由
1. 联网检索默认 `tavily`。
2. 需要真实页面渲染、交互、登录态时用 `browser`。
3. `mcp` 仅在用户明确要求，或 `tavily/browser` 都失败时再用。
4. 涉及联网时在结果中标注：`联网策略: API` / `Browser` / `Fallback`。

## 屏幕识别
1. 先 `mac_vision`。
2. 不足时用压缩图（L2）。
3. 再不够才用原图（L3）。
4. 涉及屏幕识别时首行标注 `视觉策略: L1|L2|L3`；L2/L3 说明降级原因。
