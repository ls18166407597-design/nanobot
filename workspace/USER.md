# 用户基础档案 (User Baseline)

有关老板（Boss）的持久、静态事实。

## 👤 基本身份
- **称呼**: 老板 (Boss)
- **时区**: 系统自动探测 (默认 UTC+8)
- **主要语言**: 简体中文 (Chinese-Simplified)

## 📌 沟通基准
- **风格**: 专业、干练、结果导向。
- **反馈**: 始终使用简体中文回答，反馈应简洁有力，避免过度装饰。
- **渠道自适应**: 识别交互渠道。在 Telegram 或移动端，优先采用平铺文本，避免复杂的 Markdown 表格和长嵌套。
- **报告格式**: 采用 Markdown，关键结论需加粗。

## ⚙️ 环境感知
- **工作区**: 优先使用仓库根目录下的 `workspace/`（若存在）；否则由系统动态定位。操作前需通过 `list_dir` 确认上下文。
- **联网策略**: 允许主代理直接调用联网工具，确保信息的实时性。
 
## 📂 数据索引 (Data Index)
- **通讯录数据**: `workspace/scripts/contacts.json`
  - 备注：此文件记录了所有 Telegram 和微信联系人的别名、实名及 App 映射。
- **账号与环境感知**:
  - 指令：机器人执行桌面自动化时，必须优先探测当前登录账号。如果环境反馈与指令要求账号不符，应执行切换动作并再次核实。

## 📍 指令惯例 (Conventions)
- **自动关闭**: 除非老板明确要求保留窗口，否则在桌面自动化任务完成后，**必须**使用 `--close` 参数关闭应用程序。
- **视觉优化**: 在进行视觉分析（LLM Vision）时，默认使用 `sips -Z 1024` 压缩截图，以适配 MacBook Air M2 屏幕并节省 Token。只有在识别失败或老板要求时才使用原图。
