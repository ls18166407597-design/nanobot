# Nanobot 项目结构说明 🏗️

这是 Nanobot (秘书进阶版) 的完整文件布局与逻辑分层。

## 📁 目录一览

```
nanobot/
├── .home/                   # 📄 [运行] 临时日志与持久化配置存储 (由 start.sh 指定)
│   ├── gateway.log          #    - 网关运行日志
│   └── audit.log            #    - 审计日志
│
├── nanobot/                 # ⚙️ [内核] 不可变引擎源码
│   ├── agent/               #    - 智能体循环与上下文管理
│   ├── cli/                 #    - 终端调试套件 (config, doctor, logs)
│   ├── library/             #    - 瘦身后的技能广场本地库
│   ├── tests/               #    - 核心引擎底层单元测试
│   └── maintenance/         #    - 系统维护脚本 (原 scripts/)
│
├── workspace/               # 🧠 [大脑] 活跃工作区 (你的个性化资产)
│   ├── IDENTITY.md          #    - 身份、性格与使命 (已合并 SOUL)
│   ├── AGENTS.md            #    - 执行协议与安全边界
│   ├── TOOLS.md             #    - 工具组合技手册
│   ├── scripts/             #    - 你的自动化业务脚本 (微信/电报等)
│   ├── tests/               #    - 业务与技能层面的验证测试
│   └── memory/              #    - 经整理的长短期知识库
│
├── docs/                    # 📖 [文档] 所有的指南与路线图
│   ├── 配置指南.md
│   ├── 路线图.md
│   └── 项目结构.md          #    - 本文件
│
├── ARCHITECTURE.md          # 📜 引擎与大脑隔离协议 (项目宪法)
├── README.md                # 📘 项目主入口
└── pyproject.toml           # ⚙️ 依赖与工具链配置
```

---

## 🔑 核心组件说明

| 组件 | 作用 | 修改场景 |
| :--- | :--- | :--- |
| **`nanobot/agent/loop.py`** | 系统的 CPU。 | 修改思考逻辑或错误重试机制。 |
| **`workspace/AGENTS.md`** | 系统的 执行协议。 | 改变委派规则、搜索流程 or 失败处理。 |
| **`workspace/IDENTITY.md`** | 系统的 灵魂与指令。 | 修改 AI 的性格语气、目标定位。 |
| **`ARCHITECTURE.md`** | 系统的 物理规约。 | 调整目录分层或权限边界。 |

---
*注：本项目已实现“内核稳定、大脑飞跃”的隔离架构。🐾*
