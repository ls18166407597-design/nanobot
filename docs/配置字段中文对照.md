# 配置字段中文对照（基于当前项目）

说明：`config.json` 的键名必须保持英文（代码按这些键解析），不建议改成中文键。  
可通过本文件快速理解每个字段含义与建议值。

## agents.defaults
- `workspace`：工作区目录（建议固定为项目内 `workspace`）
- `model`：默认模型名（当前：`gemini-3-flash`）
- `temperature`：生成随机度（建议 0.1~0.3）
- `maxToolIterations`：单轮最大工具迭代次数（当前 20）

## channels.telegram
- `enabled`：是否启用 Telegram 渠道
- `token`：机器人 Token
- `allowFrom`：允许的用户 ID 列表；为空表示不做白名单限制
- `proxy`：Telegram 代理（可选）

建议：
- 生产环境建议配置 `allowFrom` 白名单，避免陌生用户触发机器人。

## providers
- `providers.openai`：OpenAI 兼容入口（本项目用于本地网关时常见）
- 其他 provider（anthropic/openrouter/deepseek...）未使用时可留空

说明：
- 实际优先路由还会参考 `brain.providerRegistry`。

## tools
- `tools.exec.timeout`：`exec/task` 命令超时秒数（当前 120，适合桌面自动化）
- `tools.restrictToWorkspace`：是否限制工具只访问工作区

建议：
- 若你经常需要访问 `.home` 或系统路径，保持 `false`。
- 若更重视隔离安全且业务都在工作区，可改 `true`。

## brain
- `autoSummarize`：是否自动摘要历史上下文
- `lightRag`：是否启用轻量记忆检索
- `safetyGuard`：执行安全防护（关闭会更灵活，但风险更高）
- `memoryChunkSize`：记忆切片大小
- `summaryThreshold`：触发摘要的消息阈值
- `maxTurnSeconds`：单轮 AI 处理超时（当前 120）
- `providerRegistry`：可用模型提供商注册表（当前有 `gemini-local`、`qwen-3-8b`）

## 当前配置建议（结合你的使用场景）
1. 保持 `tools.exec.timeout=120`（已是该值）。
2. 保持 `brain.maxTurnSeconds=120`（已是该值）。
3. 若机器人仅供你自己使用，建议配置 `channels.telegram.allowFrom` 白名单。
4. 若后续强调安全，建议评估将 `brain.safetyGuard` 从 `false` 改为 `true`。
