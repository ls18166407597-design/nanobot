# Nanobot 战略路线图

本路线图概述了 Nanobot 从轻量级脚本演进为极具韧性的代理平台的长期愿景。

## ✅ 第一阶段：核心增强 (已完成)
- [x] **进阶工具套件**: Mac 控制、GitHub、知识库集成。
- [x] **语音转文字**: 通过 Groq Whisper 实现。
- [x] **精简上下文优化**: 大幅降低 Token 消耗。

## ✅ 第二阶段：智能与扩展 (已完成)
- [x] **长效记忆 (RAG)**: 实现轻量级 RAG 检索。
- [x] **并行工具执行**: 支持多工具并发调用。
- [x] **子代理并行化**: 能够生成后台子代理执行研究。

## ✅ 第三阶段：架构隔离与治理 (已完成)
- [x] **内核与大脑分离 (V2.0)**: 建立 `nanobot/` 与 `workspace/` 物理隔离架构。
- [x] **提示词工程瘦身**: 合并 `SOUL.md`，单次交互节省 1k+ Token。
- [x] **自愈机制与循环检测**: 实现基于签名的逻辑自纠偏与重复调用阻断。
- [x] **文档全量汉化**: 完成 `docs/` 目录中中文版重构。

## ✅ 第四阶段：通信时效与可靠性 (已完成)
- [x] **消息时效增强**: 历史消息注入 `[HH:MM]` 时间戳，全局对齐北京时间 (UTC+8)。
- [x] **iMessage 渠道隔离**: 实现独立的消息监听器，杜绝跨渠道消息污染。
- [x] **自动化风险排查**: 完成 Telegram 任务重复投递深度诊断。

## 🚀 下一步计划 (进行中)
- [ ] **多账号任务合并**: 重构 `tg_combined_tasks.py` 实现 UI 原子化操作。
- [ ] **深度向量 RAG**: 集成本地向量数据库。
- [ ] **UI 自动化增强**: 提升对复杂 macOS 应用的视觉点击能力。
